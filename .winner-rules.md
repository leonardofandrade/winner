# Winner — AI Development Rules

This file defines the mandatory rules that ANY AI agent (Claude Code, Cursor, Copilot, Continue.dev, etc.) must follow when working inside the Winner project.

The goal is to ensure:
- Clean Architecture
- Consistent code style
- Predictable behavior
- High-quality contributions
- Strict separation of responsibilities

---

# 🔷 1. Language & Style

- **All code MUST be written in English.**
- **All comments MUST be written in Portuguese.**
- Use modern Python 3.12+ and Django 6+ features.
- Use type hints whenever possible.
- Keep methods small and cohesive.
- Avoid duplicated logic.

---

# 🔷 2. Mandatory Architecture

Winner follows a strict clean architecture.  
All logic MUST be placed in the correct layer:

### ✔ Service Layer (business logic)
- Contains all business rules.
- Does NOT access the ORM directly.
- Uses repositories.
- May be async.

### ✔ Repository Layer (database access)
- Encapsulates ORM operations.
- Contains NO business logic.
- May be async.

### ✔ Clients (external HTTP)
- Use `httpx` (async).
- Must NOT depend on Django.
- Must NOT contain business logic.

### ✔ Parsers
- Receive raw text.
- Return Python dictionaries or lists.
- Must NOT know about Django models.

### ✔ Tasks / Commands
- Only call services.
- Must NOT contain logic.

### ✔ Views / API
- Must be thin.
- Only call services.
- No business logic.

### ✔ Models
- Simple.
- No business logic.
- No signals.
- Use Django 6+ ORM features.

---

# 🔷 3. Forbidden Practices

AI agents MUST NOT:

- Put logic in models, views, serializers, commands, or signals.
- Access the ORM directly from services.
- Write code in Portuguese.
- Create UI before Phase 8.
- Create ML before Phase 7.
- Break the architecture.
- Ignore the roadmap.
- Generate giant functions or classes.

---

# 🔷 4. Commit Rules

All commits MUST follow **Conventional Commits**, as defined in `docs/COMMIT_GUIDE.md`.

Examples:

feat(services): add Lotofacil import service
fix(repositories): correct contest update logic
refactor(parsers): simplify parsing flow
docs: update architecture guide
