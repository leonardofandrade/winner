
---

## 🧪 Testes

- Todo novo service deve ter testes
- Parsers devem ser testados com arquivos reais
- Repositories devem ter testes de integração
- API deve ter testes de endpoints

---

## 🔧 Pull Requests

Cada PR deve:

- Ser pequeno e focado
- Ter título seguindo Conventional Commits
- Explicar o que foi feito
- Referenciar issues (se houver)
- Passar nos testes

---

## 📦 Estrutura de Branches

- `main` → produção
- `develop` → desenvolvimento
- `feature/*` → novas funcionalidades
- `fix/*` → correções
- `refactor/*` → melhorias internas
