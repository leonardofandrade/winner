# Winner — Claude Project Rules

You are an AI development agent working inside the **Winner** project.  
You MUST follow all rules below for every action, suggestion, refactor, or code generation.

---

## 1. Language & Style
- All **code MUST be written in English**.
- All **comments MUST be written in Portuguese**.
- Use Python 3.12+ and Django 6+ best practices.
- Use type hints whenever possible.
- Keep methods small, cohesive, and clean.

---

## 2. Mandatory Architecture (Clean Architecture)
Winner uses a strict layered architecture.  
You MUST respect it at all times.

### ✔ Allowed logic placement:
- **Service Layer** → all business logic  
- **Repository Layer** → all ORM access  
- **Clients** → external HTTP (httpx async)  
- **Parsers** → convert raw data into Python structures  
- **Tasks / Commands** → call services only  
- **API Views** → thin, call services only  
- **Models** → simple, no logic  
- **Telegram Bot** → handlers thin, logic in services  

### ❌ Forbidden:
- Logic in models  
- Logic in views  
- Logic in serializers  
- Logic in commands  
- Logic in handlers  
- Logic in signals  
- ORM access outside repositories  
- Mixing responsibilities  

---

## 3. Roadmap Awareness
Winner follows this roadmap:

1. Setup  
2. Models & Serializers  
3. Client + Parser  
4. Import Service  
5. Result Calculation  
6. API  
7. IA (heuristics + ML prep)  
7.5 **Telegram Bot (before UI)**  
8. UI  

You MUST:
- Identify which phase the user request belongs to  
- Never advance phases unless explicitly instructed  

---

## 4. Telegram Bot Rules (Phase 7.5)
- Handlers MUST be thin  
- All bot logic MUST be in services  
- Use python-telegram-bot 21+ (async)  
- Commands MUST NOT contain logic  
- Runner MUST only start polling  
- Bot MUST integrate with existing Winner services
